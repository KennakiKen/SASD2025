# HW3: ตัวอย่าง C# Class จาก .Net Based APIs
## ตัวอย่าง Utility Class
### Class: `System.Net.WebUtility`
```csharp
using System;
using System.Net;

string original = "Hello World!";
string encoded = WebUtility.UrlEncode(original);
string decoded = WebUtility.UrlDecode(encoded);

Console.WriteLine(encoded);  // Hello+World%21
Console.WriteLine(decoded);  // Hello World!
```
System.Net.WebUtility เป็น Utility Class เพราะ Utility เป็นเหมือนกับกล่องเครื่องมือมีไว้ใช้งานในตัวได้เลย เราสามมารถ endcode กับ decode ได้โดยใช่คำสั่งของมันได้เลย
- ไม่มีสถานะ state ของตัวเอง
- เมธอดส่วนใหญ่เป็น static
- ไม่ต้อง new object
- ไม่มี field เก็บข้อมูลภายใน

## ตัวอย่าง Mutable Class
### Class: `System.Collections.Generic.List<T>`

```csharp
using System;
using System.Collections.Generic;

List<int> numbers = new List<int>();
numbers.Add(10);
numbers.Add(20);
numbers.Remove(10);

Console.WriteLine(numbers.Count);  // 1
```
System.Collections.Generic.List<T> เป็น Mutable Class เพราะสามารถเปลี่ยนแปลงข้อมูลภายใน object ได้หลังจากสร้างแล้ว
เช่น การใช้ Add(), Remove(), Clear() จะทำให้ค่าภายใน list เปลี่ยนไปโดยไม่ต้องสร้าง object ใหม่
* มี state (เก็บข้อมูลสมาชิกภายใน list)
* สามารถเปลี่ยนแปลง state ได้หลังสร้าง object
* เมธอดจะปรับค่าของ object เดิม
* ใช้ new ในการสร้าง instance

ดังนั้น List<T> จึงจัดอยู่ในกลุ่ม Mutable Class เพราะสถานะของ object สามารถเปลี่ยนแปลงได้ตลอดอายุการใช้งาน

## ตัวอย่าง Immutable Class
### Class: `System.String`
```csharp
using System;

string text = "hello";
string upper = text.ToUpper();
string replaced = text.Replace("h", "H");

Console.WriteLine(text);      // hello
Console.WriteLine(upper);     // HELLO
Console.WriteLine(replaced);  // Hello
```
System.String เป็น Immutable Class เพราะเมื่อสร้างค่าแล้วจะไม่สามารถเปลี่ยนแปลงค่าภายใน object เดิมได้

ถึงแม้จะเรียกใช้เมธอด เช่น ToUpper() หรือ Replace() จะไม่ได้แก้ไขค่าเดิม แต่จะสร้าง object ใหม่ขึ้นมาแทน

> มี state (เก็บค่าข้อความ)

> ไม่สามารถแก้ไข state เดิมได้หลังจากสร้าง

> ทุกการเปลี่ยนแปลงจะคืนค่าเป็น string ใหม่

> ค่าของตัวแปรเดิมยังคงเหมือนเดิม

ดังนั้น String จึงจัดอยู่ในกลุ่ม Immutable Class เพราะสถานะของ object ไม่สามารถเปลี่ยนแปลงได้หลังจากถูกสร้างแล้ว
