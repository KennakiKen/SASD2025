- Pattern 1: Event Notification 
### สรุปเนื้อหา
Martin พูดถึงลูกค้าที่ชอบเปลี่ยนที่อยู่ซึ่งต่างจากการเปลี่ยนพวกอีเมลที่เพิ่มช่องไปใหม่ เเต่มันต้องเปลี่ยนทั้งหมดเช่นพวกค่าที่อยู่หรืออะไรต่างๆมากมายในข้อมูลระบบต้องเเก้หลายจุดอาจเกินข้อผิดพลาดได้
เขาเลยใช้ Event Notification คือเมื่อใดก็ตามที่มีการเปลี่ยนเเปลงข้อมูลจะแจ้งเตือนให้ระบบอื่นๆ ที่เกี่ยวข้องรับรู้ถึงการเปลี่ยนแปลงนั้นระบบที่รับ event สามารถเลือกดึงข้อมูลที่ต้องการไปปรับปรุงของตนเองได้ 
วิธีนี้ช่วยลดการพึ่งพากันระหว่างระบบ ทำให้ซอฟต์แวร์มีความยืดหยุ่น ดูแลรักษาง่าย และลดโอกาสเกิดข้อผิดพลาดในระบบ ใช้ได้ทั้งระบบขนาดใหญ่เเละเล็ก
